# Управление метаданными 1С (1c-metadata-manage)

## Обзор
Навык предназначен для эффективной работы со структурой конфигурации 1С:Предприятие 8.3. Он включает правила именования, паттерны проектирования и инструкции по использованию EDT/MCP инструментов для манипуляции метаданными.

## Правила проектирования

### 1. Именование объектов
- **Префикс:** Всегда используйте префикс `ЛМ_` для новых объектов и реквизитов.
- **Язык:** Только русский язык для имен и синонимов.
- **Стиль:** Понятные бизнес-имена (например, `ЛМ_СтатусЗаказа` вместо `ЛМ_Статус`).

### 2. Структура реквизитов
- **Типы данных:** Предпочитайте ссылочные типы для связей. Для строк всегда указывайте ограничение длины (если не неограниченная).
- **Индексирование:** Индексируйте поля, которые будут использоваться в условиях `ГДЕ` запросов или для связи таблиц.

## Рабочий процесс (Workflow)

1. **Анализ:** Используйте `metadatasearch` для поиска похожих объектов в конфигурации.
2. **Проектирование:** Опишите структуру объекта (реквизиты, табличные части) перед созданием.
3. **Реализация:** Используйте `create_metadata` для верхнеуровневых объектов и `add_metadata_child` для реквизитов.
4. **Валидация:** Проверьте целостность через `edt_metadata_details`.

## Интеграция с EDT/MCP
- `create_metadata`: создание Catalog, Document, InformationRegister и др.
- `add_metadata_child`: добавление Attribute, TabularSection, Dimension, Resource.
- `update_metadata`: изменение свойств (синоним, комментарий, тип).

## Пример запроса
"Создай справочник ЛМ_Проекты с реквизитами Руководитель (СправочникСсылка.Пользователи) и ДатаНачала (Дата)."
