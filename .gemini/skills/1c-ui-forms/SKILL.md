# Разработка форм 1С (1c-ui-forms)

## Обзор
Навык помогает проектировать удобные и быстрые управляемые формы, соблюдая современные стандарты платформы и рекомендации 1С.

## Основные принципы интерфейса

### 1. Клиент-серверное взаимодействие
- **Минимизация контекстных вызовов:** Предпочитайте `&НаСервереБезКонтекста` для вызовов, не требующих данных формы.
- **Асинхронность:** КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНЫ модальные окна. Используйте `Асинх` / `Ждать` и методы `Показать...`.
- **Длительные операции:** Для тяжелых вычислений используйте механизм БСП `ДлительныеОперации`.

### 2. Структура формы
- **Команды:** Всегда выносите логику в команды формы. Группируйте кнопки в командные панели.
- **Видимость и Доступность:** Управляйте ими через функциональные опции или программно при создании формы.

## Рабочий процесс (Workflow)
1. **Проектирование:** Определите реквизиты и команды формы.
2. **Layout:** Настройте структуру элементов (группы, таблицы, поля). Используйте `mutate_form_model`.
3. **Логика:** Напишите обработчики событий (ПриСозданииНаСервере, ПриИзменении).

## Интеграция с БСП
- `СтандартныеПодсистемыКлиент.ПоказатьВопросПользователю()`: для диалогов.
- `ОбщегоНазначения.СообщитьПользователю()`: вывод сообщений в привязке к реквизиту.

## Пример запроса
"Создай форму документа с кнопкой 'Заполнить', которая асинхронно запрашивает подтверждение и выполняет заполнение на сервере без контекста."
