# Эксперт по запросам 1С (1c-query-expert)

## Обзор
Навык предоставляет лучшие практики написания высокопроизводительных запросов на языке 1С (1QL), избегая типичных ошибок и блокировок.

## Правила написания запросов

### 1. Оптимизация (Performance)
- **Запросы в циклах:** КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНЫ. Используйте пакетные запросы.
- **Временные таблицы:** Используйте `МенеджерВременныхТаблиц` для промежуточных данных. Это снижает нагрузку на TempDB.
- **Индексирование:** Всегда индексируйте поля временных таблиц, которые участвуют в соединениях (`ПО`).
- **Виртуальные таблицы:** Фильтрацию (`ГДЕ`) делайте ТОЛЬКО в параметрах виртуальной таблицы (условия на Период, Склад и т.д.).

### 2. Структура и Читаемость
- **Псевдонимы:** Всегда используйте `КАК` для всех полей и таблиц.
- **ВЫБРАТЬ *:** Запрещено. Выбирайте только необходимые поля.
- **Подзапросы:** Запрещены в секции `ВЫБРАТЬ`. Допустимы только в `ГДЕ В (...)`.

## Рабочий процесс (Workflow)
1. **Сбор данных:** Определите источники данных и необходимые поля.
2. **Параметры:** Устанавливайте параметры через `Запрос.УстановитьПараметр()`.
3. **Пакетный режим:** Группируйте запросы в один пакет, если они логически связаны.

## Интеграция с БСП
- `ОбщегоНазначения.ВыполнитьЗапросСКонтролемПрав()`: использование для безопасного выполнения.
- `ОбщегоНазначения.ЗначениеРеквизитаОбъекта()`: альтернатива простому запросу для получения 1-2 полей.

## Пример запроса
"Напиши запрос для получения остатков номенклатуры по складу 'Основной' с использованием временной таблицы для списка товаров."
