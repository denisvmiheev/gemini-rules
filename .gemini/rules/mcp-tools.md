# Справочник MCP-инструментов для разработки 1С

Все инструменты — MCP-серверы, доступные через edt-codepilot, 1c-platform, 1c-ssl, 1c-bsl-syntax-checker, 1c-workmate-checker, 1c-templates-fastcode.

## Поиск и навигация по коду (edt-codepilot)

| Инструмент | Назначение | Когда использовать |
|---|---|---|
| **grep** | Поиск текстовых паттернов в файлах проекта | Найти использования переменной, процедуры, паттерна в коде |
| **glob** | Поиск файлов по паттерну (`**/*.bsl`) | Найти файлы модулей, форм, метаданных по имени |
| **read_file** | Чтение содержимого файла (целиком или диапазон строк) | Изучить код модуля, конфигурацию метаданных |
| **bsl_list_methods** | Список процедур/функций модуля с номерами строк | Быстро изучить структуру модуля без чтения всего файла |
| **bsl_get_method_body** | Тело конкретной процедуры/функции по имени | Точечное чтение одного метода — экономия контекста |
| **edt_find_references** | Семантический поиск всех ссылок на объект метаданных | Найти все места использования справочника, документа, реквизита |
| **bsl_scope_members** | Доступные методы/свойства в позиции кода | Узнать какие методы можно вызвать у объекта |
| **bsl_symbol_at_position** | Определение символа (метод, переменная) в позиции | Понять что за сущность в конкретном месте |
| **bsl_type_at_position** | Определение типа выражения в позиции | Узнать тип переменной без запуска |

## Редактирование (edt-codepilot)

| Инструмент | Назначение | Когда использовать |
|---|---|---|
| **edit_file** | Редактирование файла (search/replace или полная замена) | Модификация существующего кода. **НЕ для .mdo файлов!** |
| **write_file** | Запись содержимого в существующий файл | Запись нового кода в подготовленный файл. **НЕ для .mdo!** |

## Метаданные (edt-codepilot)

| Инструмент | Назначение | Когда использовать |
|---|---|---|
| **scan_metadata_index** | Поиск по структуре метаданных конфигурации | Проверка существования объектов, атрибутов, связей |
| **edt_metadata_details** | Детальная информация об объекте метаданных | Получение полной структуры конкретного объекта |
| **create_metadata** | Создание нового объекта метаданных верхнего уровня | Создание справочника, документа, регистра |
| **add_metadata_child** | Создание вложенного объекта (реквизит, ТЧ, форма) | Добавление реквизитов, табличных частей, команд |
| **ensure_module_artifact** | Гарантирует наличие файла модуля (.bsl) для объекта | Перед записью кода модуля объекта/менеджера |
| **edt_validate_request** | Проверка мутации и получение validation_token | **Обязательно** перед любой мутацией метаданных |

## Управляемые формы (edt-codepilot)

| Инструмент | Назначение | Когда использовать |
|---|---|---|
| **create_form** | Создание новой управляемой формы | Создание формы объекта/списка/выбора/вспомогательной |
| **inspect_form_layout** | Чтение структуры формы (дерево элементов, id, dataPath, свойства) | Изучение существующей формы перед модификацией |
| **mutate_form_model** | Изменение модели формы (добавление/удаление/перемещение элементов) | Программная модификация макета формы |
| **apply_form_recipe** | Декларативный рецепт: создание формы + реквизиты + макет за один вызов | Комплексное создание/обновление формы целиком |

### Workflow работы с формами

#### Изучение существующей формы
1. **scan_metadata_index** — найти форму по имени объекта
2. **inspect_form_layout** — получить дерево элементов формы (id, dataPath, свойства)
3. **bsl_list_methods** — изучить структуру модуля формы
4. **bsl_get_method_body** — прочитать конкретные обработчики

#### Модификация существующей формы
1. **inspect_form_layout** — изучить текущую структуру
2. **edt_validate_request** — получить validation_token для мутации
3. **mutate_form_model** — добавить/изменить/удалить элементы формы
4. **edit_file** — модифицировать модуль формы (BSL-код обработчиков)
5. **syntaxcheck** — проверить синтаксис модуля

#### Создание новой формы
1. **edt_validate_request** — получить validation_token
2. **apply_form_recipe** — создать форму с реквизитами и макетом за один вызов
   - Или пошагово: **create_form** → **mutate_form_model**
3. **ensure_module_artifact** — гарантировать наличие файла модуля формы
4. **write_file** — записать код модуля формы
5. **syntaxcheck** — проверить синтаксис

#### Операции mutate_form_model
| Операция | Назначение |
|---|---|
| `set_form_props` | Установка свойств формы (заголовок, размеры) |
| `add_group` | Добавление группы элементов (страница, панель, колонка) |
| `add_field` | Добавление поля формы (ввод, флажок, надпись) |
| `set_item` | Изменение свойств существующего элемента |
| `remove_item` | Удаление элемента формы |
| `move_item` | Перемещение элемента в другую группу/позицию |

### Важные правила работы с формами
- **НЕ редактировать .mdo файлы напрямую** через edit_file/write_file — использовать MCP-инструменты метаданных
- **Всегда получать validation_token** через `edt_validate_request` перед мутацией
- **Всегда inspect_form_layout** перед модификацией — понять текущую структуру
- Для модуля формы: `ensure_module_artifact` → `write_file` / `edit_file`
- Типовые формы модифицировать **программно** (через код в ЛМ_ПодготовитьФорму), а не визуально

## Документация и шаблоны

| Инструмент | Назначение | Когда использовать |
|---|---|---|
| **docsearch** | Поиск по документации платформы 1С | Проверка существования встроенных функций, сигнатуры методов |
| **templatesearch** | Поиск шаблонов и примеров кода FastCode | Найти архитектурные паттерны, примеры перед написанием кода |
| **ssl_search** | Поиск функций БСП | Найти стандартные функции библиотеки для переиспользования |

## Качество кода

| Инструмент | Назначение | Когда использовать |
|---|---|---|
| **syntaxcheck** | Проверка синтаксиса и стиля BSL | После написания кода. **Лимит: 3 раза за цикл** |
| **check_1c_code** | Анализ логических и производительных проблем | Поиск проблем производительности, логических ошибок |

## Workflow по задачам

### Написание нового кода
1. **templatesearch** — найти похожие реализации
2. **grep** — найти существующие паттерны в проекте
3. **scan_metadata_index** — проверить структуру метаданных
4. **docsearch** — проверить встроенные функции
5. **ssl_search** — найти переиспользуемые функции БСП
6. **edit_file** / **write_file** — написать код
7. **syntaxcheck** — проверить синтаксис после написания
8. **check_1c_code** — проверить на проблемы

### Изучение существующего кода
1. **glob** — найти файлы модулей по паттерну
2. **bsl_list_methods** — изучить структуру модуля (методы, строки, параметры)
3. **bsl_get_method_body** — прочитать конкретный метод
4. **edt_find_references** — найти все использования объекта
5. **bsl_scope_members** — понять доступные методы в контексте

### Ревью кода
1. **grep** — проверить соответствие паттернам
2. **scan_metadata_index** — проверить корректность использования метаданных
3. **docsearch** — проверить существование методов/свойств
4. **check_1c_code** — автоматический анализ

### Исправление ошибок
1. **syntaxcheck** — определить синтаксические ошибки
2. **docsearch** — проверить правильные имена функций
3. **scan_metadata_index** — проверить имена объектов метаданных
4. **grep** — найти правильные паттерны использования
5. **edit_file** — исправить код

### Рефакторинг
1. **bsl_list_methods** — изучить структуру модуля
2. **edt_find_references** — найти все использования рефакторируемого кода
3. **grep** — дополнительный поиск динамических вызовов
4. **edit_file** — применить изменения
5. **syntaxcheck** — проверить результат

### Оптимизация производительности
1. **grep** — найти проблемные паттерны (запросы в циклах, точечная нотация)
2. **scan_metadata_index** — проверить индексацию
3. **check_1c_code** — автоматический анализ производительности
4. **templatesearch** — найти оптимизированные паттерны

## Важные правила

1. **Всегда ищи перед написанием** — используй templatesearch и grep сначала
2. **Проверяй по документации** — используй docsearch для встроенных функций
3. **Проверяй существование метаданных** — используй scan_metadata_index перед использованием объектов
4. **Лимит syntaxcheck** — максимум 3 раза за цикл, затем двигайся дальше
5. **Предпочитай MCP-инструменты** — `grep`/`glob`/`read_file` вместо встроенных `Grep`/`Glob`/`Read`
6. **Семантика вместо текста** — `edt_find_references` вместо grep по имени объекта
7. **Точечное чтение** — `bsl_list_methods` + `bsl_get_method_body` вместо чтения всего модуля
8. **ЗАПРЕЩЕНО** использовать **get_diagnostics**
