# Стандарты разработки: Ядро

## Стиль кода

### Отступы и форматирование
- **TAB** для отступов (не пробелы)
- Максимальная длина строки: **120 символов**
- Одна инструкция на строку
- Пустая строка перед `Возврат` (если `Возврат` не единственная инструкция)
- Пустые строки для визуального разделения логических блоков

### Именование
- Переменные — `CamelCase`, семантически отражают тип: `МассивСчетов`, `ТаблицаТоваров`
- Булевы — положительные: `Разрешено`, `ПроверкаПройдена` (не `НеЗапрещено`, `ПроверкаНеПройдена`)
- Запрещены имена из глобального контекста: `Документы`, `Справочники`, `Метаданные`, `Константы`
- «Магические числа» запрещены — выносить в именованные переменные

### Операторы и выражения
- Тернарный оператор `?()` **ЗАПРЕЩЕН**
- Сравнения с `Истина`/`Ложь` запрещены, использовать булево напрямую
- Предпочитать `СтрШаблон()` для составления строк
- Цепочное обращение (`Запрос.Выполнить().Выгрузить()`) запрещено

### Выравнивание
- Для групп присваиваний локальных переменных — выравнивать `=` пробелами
- **НЕ** выравнивать при установке свойств объекта через точку

## Комментарии модификаций

### Формат обрамления
```bsl
// ++ https://tracker.yandex.ru/%ИмяВетки%
<Измененный или добавленный код>
// -- https://tracker.yandex.ru/%ИмяВетки%
```

### Удаление типового кода
```bsl
// ++ https://tracker.yandex.ru/%ИмяВетки%
// Закомментированный типовой код
// ОригинальнаяСтрока1;
// ОригинальнаяСтрока2;
<Новый код замены (если есть)>
// -- https://tracker.yandex.ru/%ИмяВетки%
```

**Типовой код НИКОГДА не удаляется — только закомментируется.**

### Новые процедуры/функции
```bsl
// ++ https://tracker.yandex.ru/%ИмяВетки%
Процедура ЛМ_НоваяПроцедура(Параметры)
    // Тело процедуры
КонецПроцедуры // -- https://tracker.yandex.ru/%ИмяВетки%
```

## Документирование процедур (стандарт БСП)

```bsl
// Описание назначения процедуры/функции.
// Продолжение описания при необходимости.
//
// Параметры:
//  ИмяПараметра1 - Тип - описание параметра.
//  ИмяПараметра2 - Тип - описание параметра.
//                  Продолжение описания параметра.
//
// Возвращаемое значение:
//  Тип - описание возвращаемого значения.
//
Функция ИмяФункции(ИмяПараметра1, ИмяПараметра2) Экспорт
```

### Правила документирования
- Описание — что делает (не как)
- Пустая строка между описанием и параметрами
- Каждый параметр: `ИмяПараметра - Тип - описание`
- Возвращаемое значение только для функций
- `Экспорт` — обязательно для публичного API

## Метрики качества

| Метрика | Лимит | Строгость |
|---|---|---|
| Длина метода | ≤ 200 строк | жесткий |
| Длина метода | > 100 строк | триггер ревью |
| Вложенность | < 5 уровней | жесткий |
| Когнитивная сложность | < 15 | триггер ревью |
| Параметры | ≤ 5 | жесткий |
| Длина строки | ≤ 120 символов | жесткий |

## Выбор типа объекта метаданных

| Задача | Тип объекта |
|--------|------------|
| Одно значение на всю ИБ | Константа |
| Классификатор без иерархии | Перечисление |
| Справочные данные с иерархией | Справочник |
| Бизнес-операция, регистрирующая движения | Документ |
| Срезы данных на дату | Регистр сведений (периодический) |
| Учет остатков (приход/расход) | Регистр накопления (остатки) |
| Учет оборотов | Регистр накопления (обороты) |
| Периодическая обработка | Регламентное задание |
| Переиспользуемая бизнес-логика | Общий модуль |
| Права доступа | Роль |
