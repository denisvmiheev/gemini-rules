# Стандарты разработки: Архитектура

## Архитектурные принципы

### Разделение ответственностей
- **Модуль объекта / менеджера** — бизнес-логика предметной области
- **Модуль формы** — только UI-логика и подготовка данных для отображения
- **Общий модуль** — переиспользуемая логика, вызываемая из нескольких мест
- **Модуль команды** — точка входа для команд интерфейса

### Слои приложения
```
Клиент → Серверные процедуры формы → Общие модули → Данные (запросы, объекты)
```

- Клиент: UI-взаимодействие, навигация, вызовы сервера
- Сервер формы: подготовка данных формы, валидация
- Общие модули: бизнес-логика, расчеты, интеграции
- Данные: запросы, чтение/запись объектов

### Директивы компиляции

| Директива | Назначение | Когда использовать |
|---|---|---|
| `&НаКлиенте` | Клиентский код | UI-взаимодействие, диалоги, навигация |
| `&НаСервере` | Серверный с контекстом формы | Когда нужен доступ к данным формы |
| `&НаСервереБезКонтекста` | Серверный без контекста | **Приоритет!** Когда контекст не нужен |
| `&НаКлиентеНаСервереБезКонтекста` | Общий клиент-сервер | Общие проверки и расчеты |

**Правило:** Всегда предпочитать `&НаСервереБезКонтекста`. Использовать `&НаСервере` только если реально нужен контекст формы.

## Общие модули

### Типы общих модулей

| Суффикс | Контекст | Назначение |
|---|---|---|
| (без суффикса) | Сервер | Основная серверная бизнес-логика |
| `Клиент` | Клиент | Клиентская логика |
| `КлиентСервер` | Клиент + Сервер | Общие функции (валидация, форматирование) |
| `ВызовСервера` | Сервер (вызов с клиента) | Тонкая обертка для клиентских вызовов |
| `ПовтИсп` | Сервер + кэш | Кэшируемые данные (на время вызова или сеанса) |
| `Переопределяемый` | Сервер | Точки расширения для переопределения |
| `Глобальный` | Клиент (глобальный) | Глобальные обработчики событий |
| `ЛокализацияСервер` | Сервер | Локализованная бизнес-логика |
| `ЛокализацияКлиент` | Клиент | Локализованная клиентская логика |

### Правила общих модулей
- Модуль `ВызовСервера` — **только тонкая обертка**, без бизнес-логики
- `ПовтИсп` — только чтение, **НИКОГДА не модифицировать данные**
- Экспортные процедуры — публичный API, документировать по стандарту БСП
- Неэкспортные — внутренняя реализация

## Расширения конфигурации

### Когда использовать расширения
- Небольшие доработки, не затрагивающие ядро
- Добавление новых объектов метаданных без изменения типовых
- Переопределение поведения через подписки на события
- Исправления, которые не должны конфликтовать с обновлениями типовой конфигурации

### Правила расширений
- Расширение **НЕ** должно дублировать объекты основной конфигурации целиком
- Заимствованные модули — минимальные изменения, с обрамлением комментариями
- Собственные объекты расширения — с префиксом, без дублирования логики
- Подписки на события — предпочтительный способ расширения поведения

## Паттерны проектирования для 1С

### Фасад (Общий модуль как API)
Общий модуль предоставляет публичный API, скрывая внутреннюю реализацию:
```
МодульВызовСервера → МодульСервер → Внутренние процедуры
```

### Стратегия (через Соответствие)
```bsl
ОбработчикиСобытий = Новый Соответствие;
ОбработчикиСобытий.Вставить("Приход", "ОбработатьПриход");
ОбработчикиСобытий.Вставить("Расход", "ОбработатьРасход");

ИмяОбработчика = ОбработчикиСобытий.Получить(ТипОперации);
Если ИмяОбработчика <> Неопределено Тогда
    Выполнить(ИмяОбработчика + "(Параметры)");
КонецЕсли;
```

### Построитель (для сложных запросов)
```bsl
СхемаЗапроса = Новый СхемаЗапроса;
СхемаЗапроса.УстановитьТекстЗапроса(БазовыйТекстЗапроса);
// Модификация схемы программно
ТекстЗапроса = СхемаЗапроса.ПолучитьТекстЗапроса();
```

## Code Smells для 1С

### Признаки проблем
- Метод > 200 строк — декомпозировать
- > 5 параметров — использовать Структуру
- Вложенность > 4 — ранний возврат / Продолжить
- Дублирование кода в 3+ местах — вынести в общий модуль
- `&НаСервере` без обращения к данным формы — заменить на `&НаСервереБезКонтекста`
- Серверный вызов в цикле на клиенте — объединить в один вызов
- `Попытка...Исключение` без записи в журнал — добавить логирование или убрать

### Запрещенные практики
- Использование `Выполнить()` и `Вычислить()` без крайней необходимости
- Прямые запросы к служебным таблицам БСП
- Модификация данных в модулях с повторным использованием
- Обращение к метаданным напрямую для получения бизнес-данных
