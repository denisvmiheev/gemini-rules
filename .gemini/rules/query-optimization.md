# Оптимизация запросов 1С

## Базовые правила

1. **Всегда** псевдонимы таблиц и полей (`КАК`)
2. **Запрещено** `ВЫБРАТЬ *` — перечислять нужные поля
3. **Запрещены** подзапросы в секции `ВЫБРАТЬ`
4. **Запрещены** условия `ГДЕ` на виртуальных таблицах — использовать параметры
5. **Запрещены** запросы в циклах — использовать временные таблицы или `В (&Массив)`
6. **Запрещены** цепочки: `Запрос.Выполнить().Выгрузить()`

## Паттерн: Пакетные запросы с временными таблицами

Для сложной логики — разбивать на последовательные запросы через `МенеджерВременныхТаблиц`:

```bsl
МенеджерВТ = Новый МенеджерВременныхТаблиц;

Запрос = Новый Запрос;
Запрос.МенеджерВременныхТаблиц = МенеджерВТ;
Запрос.Текст =
"ВЫБРАТЬ
|    Номенклатура.Ссылка КАК Номенклатура,
|    Номенклатура.ВидНоменклатуры КАК ВидНоменклатуры
|ПОМЕСТИТЬ ВТ_Номенклатура
|ИЗ
|    Справочник.Номенклатура КАК Номенклатура
|ГДЕ
|    Номенклатура.Ссылка В (&МассивНоменклатуры)
|;
|
|ВЫБРАТЬ
|    Остатки.Номенклатура КАК Номенклатура,
|    Остатки.КоличествоОстаток КАК Остаток
|ИЗ
|    РегистрНакопления.ТоварыНаСкладах.Остатки(
|        ,
|        Номенклатура В (ВЫБРАТЬ ВТ.Номенклатура ИЗ ВТ_Номенклатура КАК ВТ)
|    ) КАК Остатки";

Запрос.УстановитьПараметр("МассивНоменклатуры", МассивНоменклатуры);
РезультатыПакета = Запрос.ВыполнитьПакет();
ТаблицаОстатков = РезультатыПакета[1].Выгрузить();
```

## Паттерн: Оптимизация виртуальных таблиц

### Регистры накопления — Остатки
```bsl
// ПЛОХО — фильтр снаружи
"ВЫБРАТЬ Остатки.КоличествоОстаток КАК Остаток
|ИЗ РегистрНакопления.ТоварыНаСкладах.Остатки() КАК Остатки
|ГДЕ Остатки.Склад = &Склад"

// ПРАВИЛЬНО — фильтр внутри параметров ВТ
"ВЫБРАТЬ
|    Остатки.Номенклатура КАК Номенклатура,
|    Остатки.КоличествоОстаток КАК Остаток
|ИЗ
|    РегистрНакопления.ТоварыНаСкладах.Остатки(
|        &Период,
|        Склад = &Склад И Номенклатура В (&МассивНоменклатуры)
|    ) КАК Остатки"
```

### Регистры сведений — СрезПоследних
```bsl
// ПРАВИЛЬНО — условие внутри параметров
"ВЫБРАТЬ
|    Срез.Номенклатура КАК Номенклатура,
|    Срез.Цена КАК Цена
|ИЗ
|    РегистрСведений.ЦеныНоменклатуры.СрезПоследних(
|        &Дата,
|        Номенклатура В (&МассивНоменклатуры)
|            И ТипЦен = &ТипЦен
|    ) КАК Срез"
```

## Паттерн: Замена подзапросов соединениями

```bsl
// ПЛОХО — подзапрос в ВЫБРАТЬ
"ВЫБРАТЬ
|    Т.Номенклатура,
|    (ВЫБРАТЬ МАКСИМУМ(Ц.Цена)
|     ИЗ РегистрСведений.Цены КАК Ц
|     ГДЕ Ц.Номенклатура = Т.Номенклатура) КАК МаксЦена
|ИЗ Таблица КАК Т"

// ПРАВИЛЬНО — через соединение
"ВЫБРАТЬ
|    Т.Номенклатура КАК Номенклатура,
|    ЕСТЬNULL(Цены.МаксЦена, 0) КАК МаксЦена
|ИЗ
|    Таблица КАК Т
|        ЛЕВОЕ СОЕДИНЕНИЕ (
|            ВЫБРАТЬ
|                Ц.Номенклатура КАК Номенклатура,
|                МАКСИМУМ(Ц.Цена) КАК МаксЦена
|            ИЗ
|                РегистрСведений.Цены КАК Ц
|            СГРУППИРОВАТЬ ПО
|                Ц.Номенклатура
|        ) КАК Цены
|        ПО Т.Номенклатура = Цены.Номенклатура"
```

## Паттерн: ЕСТЬNULL для левых соединений

**Всегда** оборачивать поля из правой таблицы в `ЕСТЬNULL()`:
```bsl
"ВЫБРАТЬ
|    Товары.Номенклатура КАК Номенклатура,
|    ЕСТЬNULL(Остатки.КоличествоОстаток, 0) КАК Остаток
|ИЗ
|    ВТ_Товары КАК Товары
|        ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.ТоварыНаСкладах.Остатки КАК Остатки
|        ПО Товары.Номенклатура = Остатки.Номенклатура"
```

## Паттерн: Индексация временных таблиц

При соединении временных таблиц — индексировать поля соединения:
```bsl
"ВЫБРАТЬ
|    Данные.Номенклатура КАК Номенклатура,
|    Данные.Склад КАК Склад,
|    Данные.Количество КАК Количество
|ПОМЕСТИТЬ ВТ_Данные
|ИЗ
|    &ТаблицаДанных КАК Данные
|
|ИНДЕКСИРОВАТЬ ПО
|    Номенклатура,
|    Склад"
```

## Паттерн: ПЕРВЫЕ N для проверки существования

```bsl
// Для проверки "существует ли хотя бы одна запись"
"ВЫБРАТЬ ПЕРВЫЕ 1
|    ИСТИНА КАК Существует
|ИЗ
|    Документ.РеализацияТоваров КАК Реализация
|ГДЕ
|    Реализация.Контрагент = &Контрагент
|    И Реализация.Проведен"
```

## Паттерн: Агрегация вместо перебора

```bsl
// ПЛОХО — перебор в коде
Сумма = 0;
Для Каждого Строка Из ТЧТовары Цикл
    Сумма = Сумма + Строка.Сумма;
КонецЦикла;

// ПРАВИЛЬНО — агрегация в запросе
"ВЫБРАТЬ
|    СУММА(Товары.Сумма) КАК ИтогоСумма
|ИЗ
|    Документ.Реализация.Товары КАК Товары
|ГДЕ
|    Товары.Ссылка = &Ссылка"
```

Или через метод таблицы значений:
```bsl
Сумма = ТЧТовары.Итог("Сумма");
```

## Паттерн: Оптимизация условий соединения

- Предпочитать `ВНУТРЕННЕЕ СОЕДИНЕНИЕ` когда обе стороны гарантированно существуют
- `ЛЕВОЕ СОЕДИНЕНИЕ` — когда правая часть может отсутствовать
- Условие соединения — **всегда по индексированным полям** (измерения регистров, ссылки)
- Не использовать функции в условиях соединения (например, `НАЧАЛОПЕРИОДА()` в `ПО`)

## Паттерн: Обход блокировок

Для отчетов и чтения данных без блокировки:
```bsl
Запрос = Новый Запрос;
Запрос.Текст = "...";
Запрос.УстановитьПараметр("Параметр", Значение);

// Чтение без блокировки (для отчетов)
Запрос.УстановитьПараметр("РежимБлокировки", РежимБлокировкиДанных.Автоматический);
```

## Чек-лист оптимизации запроса

- [ ] Нет `ВЫБРАТЬ *` — только нужные поля
- [ ] Нет подзапросов в `ВЫБРАТЬ` — используются соединения
- [ ] Виртуальные таблицы — фильтры в параметрах, не в `ГДЕ`
- [ ] `ЕСТЬNULL()` для полей из `ЛЕВОГО СОЕДИНЕНИЯ`
- [ ] Временные таблицы проиндексированы по полям соединения
- [ ] Нет запросов в цикле — используются пакетные запросы
- [ ] `ПЕРВЫЕ N` где нужно ограниченное количество
- [ ] Условия фильтрации — по индексированным полям в первую очередь
- [ ] Нет функций в условиях соединения
- [ ] Промежуточная переменная для результата запроса
