# Каталог антипаттернов 1С

## Уровни критичности

- **CRITICAL** — нарушение приводит к серьезным проблемам производительности или данных. Немедленное исправление.
- **HIGH** — значительное влияние на производительность или поддерживаемость. Исправление при первой возможности.
- **MEDIUM** — умеренное влияние. Исправление в рамках рефакторинга.

---

## CRITICAL: Запросы в цикле

**Проблема:** Каждая итерация = обращение к СУБД. O(n) вызовов.

```bsl
// ПЛОХО
Для Каждого Элемент Из МассивСсылок Цикл
    Запрос = Новый Запрос;
    Запрос.Текст = "ВЫБРАТЬ Наименование ИЗ Справочник.Номенклатура ГДЕ Ссылка = &Ссылка";
    Запрос.УстановитьПараметр("Ссылка", Элемент);
    Результат = Запрос.Выполнить();
КонецЦикла;

// ПРАВИЛЬНО
Запрос = Новый Запрос;
Запрос.Текст =
"ВЫБРАТЬ
|    Номенклатура.Ссылка КАК Ссылка,
|    Номенклатура.Наименование КАК Наименование
|ИЗ
|    Справочник.Номенклатура КАК Номенклатура
|ГДЕ
|    Номенклатура.Ссылка В (&МассивСсылок)";
Запрос.УстановитьПараметр("МассивСсылок", МассивСсылок);
РезультатЗапроса = Запрос.Выполнить();
```

## CRITICAL: Точечная нотация через ссылку

**Проблема:** Каждое обращение через точку = неявный запрос к СУБД.

```bsl
// ПЛОХО
ИНН = Контрагент.ИНН;
Наименование = Контрагент.Наименование;

// ПРАВИЛЬНО
РеквизитыКонтрагента = ОбщегоНазначения.ЗначенияРеквизитовОбъекта(
    Контрагент, "ИНН, Наименование");
ИНН = РеквизитыКонтрагента.ИНН;
Наименование = РеквизитыКонтрагента.Наименование;
```

## CRITICAL: Подзапрос в секции ВЫБРАТЬ

**Проблема:** Подзапрос выполняется для каждой строки результата.

```bsl
// ПЛОХО
"ВЫБРАТЬ
|    Контрагенты.Ссылка,
|    (ВЫБРАТЬ СУММА(Продажи.Сумма)
|     ИЗ РегистрНакопления.Продажи КАК Продажи
|     ГДЕ Продажи.Контрагент = Контрагенты.Ссылка) КАК СуммаПродаж
|ИЗ Справочник.Контрагенты КАК Контрагенты"

// ПРАВИЛЬНО
"ВЫБРАТЬ
|    Контрагенты.Ссылка КАК Ссылка,
|    ЕСТЬNULL(ПродажиИтоги.СуммаПродаж, 0) КАК СуммаПродаж
|ИЗ
|    Справочник.Контрагенты КАК Контрагенты
|        ЛЕВОЕ СОЕДИНЕНИЕ (
|            ВЫБРАТЬ
|                Продажи.Контрагент КАК Контрагент,
|                СУММА(Продажи.Сумма) КАК СуммаПродаж
|            ИЗ
|                РегистрНакопления.Продажи КАК Продажи
|            СГРУППИРОВАТЬ ПО
|                Продажи.Контрагент
|        ) КАК ПродажиИтоги
|        ПО Контрагенты.Ссылка = ПродажиИтоги.Контрагент"
```

## HIGH: Условие ГДЕ на виртуальной таблице

**Проблема:** Виртуальная таблица материализуется целиком, потом фильтруется.

```bsl
// ПЛОХО
"ВЫБРАТЬ Остатки.Номенклатура
|ИЗ РегистрНакопления.ТоварыНаСкладах.Остатки() КАК Остатки
|ГДЕ Остатки.Склад = &Склад"

// ПРАВИЛЬНО — фильтр через параметры виртуальной таблицы
"ВЫБРАТЬ Остатки.Номенклатура КАК Номенклатура
|ИЗ РегистрНакопления.ТоварыНаСкладах.Остатки(, Склад = &Склад) КАК Остатки"
```

## HIGH: Избыточные клиент-серверные вызовы

**Проблема:** Каждый серверный вызов — маршалинг, сеть, десериализация.

```bsl
// ПЛОХО — 3 серверных вызова
&НаКлиенте
Процедура ОбработатьДанные()

    Данные = ПолучитьДанныеСервер();
    Результат = ОбработатьСервер(Данные);
    СохранитьСервер(Результат);

КонецПроцедуры

// ПРАВИЛЬНО — 1 серверный вызов
&НаКлиенте
Процедура ОбработатьДанные()

    ВыполнитьОбработкуСервер();

КонецПроцедуры

&НаСервереБезКонтекста
Процедура ВыполнитьОбработкуСервер()

    Данные = ПолучитьДанные();
    Результат = Обработать(Данные);
    Сохранить(Результат);

КонецПроцедуры
```

## HIGH: &НаСервере вместо &НаСервереБезКонтекста

**Проблема:** Контекст формы передается на сервер и обратно — сериализация всех данных формы.

```bsl
// ПЛОХО — передает весь контекст формы
&НаСервере
Функция ПолучитьКурсВалюты(Валюта, Дата)
    // ... данные формы не нужны
КонецФункции

// ПРАВИЛЬНО — без контекста, параметрами
&НаСервереБезКонтекста
Функция ПолучитьКурсВалюты(Валюта, Дата)
    // ... работает быстрее
КонецФункции
```

## MEDIUM: Отсутствие кэширования повторных вычислений

```bsl
// ПЛОХО — повторные обращения за одними данными
Для Каждого Строка Из ТЧТовары Цикл

    СтавкаНДС = ПолучитьСтавкуНДС(Строка.Номенклатура);

КонецЦикла;

// ПРАВИЛЬНО — с кэшем
КэшСтавок = Новый Соответствие;
Для Каждого Строка Из ТЧТовары Цикл

    СтавкаНДС = КэшСтавок.Получить(Строка.Номенклатура);
    Если СтавкаНДС = Неопределено Тогда

        СтавкаНДС = ПолучитьСтавкуНДС(Строка.Номенклатура);
        КэшСтавок.Вставить(Строка.Номенклатура, СтавкаНДС);

    КонецЕсли;

КонецЦикла;
```

## MEDIUM: O(n²) алгоритмы поиска

```bsl
// ПЛОХО — поиск перебором, O(n * m)
Для Каждого Элемент Из Коллекция1 Цикл
    Для Каждого Элемент2 Из Коллекция2 Цикл
        Если Элемент.Ключ = Элемент2.Ключ Тогда
            // ...
        КонецЕсли;
    КонецЦикла;
КонецЦикла;

// ПРАВИЛЬНО — индекс через Соответствие, O(n + m)
Индекс = Новый Соответствие;
Для Каждого Элемент2 Из Коллекция2 Цикл
    Индекс.Вставить(Элемент2.Ключ, Элемент2);
КонецЦикла;

Для Каждого Элемент Из Коллекция1 Цикл

    НайденныйЭлемент = Индекс.Получить(Элемент.Ключ);
    Если НайденныйЭлемент <> Неопределено Тогда
        // ...
    КонецЕсли;

КонецЦикла;
```

## MEDIUM: Глубокая вложенность (>4 уровней)

```bsl
// ПЛОХО — 5 уровней
Если Условие1 Тогда
    Если Условие2 Тогда
        Для Каждого Строка Из Таблица Цикл
            Если Условие3 Тогда
                Если Условие4 Тогда
                    // Логика
                КонецЕсли;
            КонецЕсли;
        КонецЦикла;
    КонецЕсли;
КонецЕсли;

// ПРАВИЛЬНО — ранний возврат
Если НЕ Условие1 Тогда
    Возврат;
КонецЕсли;

Если НЕ Условие2 Тогда
    Возврат;
КонецЕсли;

Для Каждого Строка Из Таблица Цикл

    Если НЕ Условие3 Тогда
        Продолжить;
    КонецЕсли;

    Если НЕ Условие4 Тогда
        Продолжить;
    КонецЕсли;

    // Логика

КонецЦикла;
```

## Архитектурные антипаттерны

### Big Ball of Mud
Модули без четкой структуры. Процедуры вне областей. Всё в одном модуле.
**Решение:** Четкое разделение на области, вынос в общие модули.

### God Module
Один модуль отвечает за всё — бизнес-логику, UI, интеграции, расчеты.
**Решение:** Разделение ответственностей. Менеджер — бизнес-логика, форма — UI, общий модуль — переиспользуемая логика.

### Tight Coupling
Жесткие зависимости между модулями через прямые вызовы.
**Решение:** Слабая связность через параметры, события, общие модули-посредники.

### Copy-Paste Architecture
Дублирование одного и того же кода в нескольких местах.
**Решение:** Вынос в общие модули, параметризация различий.

## Confidence Scoring при ревью

При обнаружении антипаттерна — указывать уверенность:

| Диапазон | Значение | Действие |
|---|---|---|
| 90-100 | Однозначный антипаттерн | Немедленное исправление |
| 70-89 | Вероятный антипаттерн | Рекомендация к исправлению |
| 50-69 | Возможный антипаттерн | Предупреждение, анализ контекста |
| <50 | Маловероятно | Информирование |
