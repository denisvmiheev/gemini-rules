## Правила модификации типовых форм

### Процедура ЛМ_ПодготовитьФорму
Все доработки интерфейса формы собираются в одну процедуру:

```bsl
#Область ОбработчикиСобытийФормы

&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)

    // ++ https://tracker.yandex.ru/ONESCIS-XXXX
    ЛМ_ПодготовитьФорму();
    // -- https://tracker.yandex.ru/ONESCIS-XXXX

    // ... типовой код ...

КонецПроцедуры

#КонецОбласти

// ... весь типовой код ...

#Область Ламода

&НаСервере
Процедура ЛМ_ПодготовитьФорму()

    // Настройка видимости, доступности, значений по умолчанию

КонецПроцедуры

// Другие процедуры доработок

#КонецОбласти
```

### Правила модификации
1. Вызов `ЛМ_ПодготовитьФорму()` — **в начале** обработчика `ПриСозданииНаСервере`
2. Все новые элементы формы — с префиксом `ЛМ_`
3. Все новые реквизиты формы — с префиксом `ЛМ_`
4. Все новые команды формы — с префиксом `ЛМ_`
5. Обработчики новых элементов — в области `#Область Ламода`

## Обработчики событий

### Добавление обработчиков событий к типовым элементам
```bsl
// В ЛМ_ПодготовитьФорму():
Элементы.ИмяЭлемента.УстановитьДействие("ПриИзменении", "ЛМ_ИмяЭлементаПриИзменении");

// Сам обработчик — в области Ламода:
&НаКлиенте
Процедура ЛМ_ИмяЭлементаПриИзменении(Элемент)
    // Логика обработки
КонецПроцедуры
```

### Расположение обработчиков по директивам
```
&НаКлиенте              → UI-реакции, оповещения, открытие форм
&НаСервере              → Изменение данных формы, условная видимость
&НаСервереБезКонтекста  → Получение данных, вычисления без контекста формы
```

## Программное управление формой

### Видимость и доступность
```bsl
// Управление видимостью элемента
Элементы.ИмяЭлемента.Видимость = УсловиеВидимости;

// Управление доступностью
Элементы.ИмяЭлемента.Доступность = УсловиеДоступности;

// Управление только чтением (для полей ввода)
Элементы.ИмяЭлемента.ТолькоЧтение = УсловиеТолькоЧтение;
```

### Программное добавление элементов
```bsl
НовыйЭлемент = Элементы.Добавить("ЛМ_ИмяЭлемента", Тип("ПолеФормы"), Элементы.ГруппаРодитель);
НовыйЭлемент.Вид = ВидПоляФормы.ПолеВвода;
НовыйЭлемент.ПутьКДанным = "Объект.ЛМ_ИмяРеквизита";
НовыйЭлемент.Заголовок = "Заголовок поля";
```

### Условное оформление
```bsl
ЭлементУО = УсловноеОформление.Элементы.Добавить();

ПолеОформления = ЭлементУО.Поля.Элементы.Добавить();
ПолеОформления.Поле = Новый ПолеКомпоновкиДанных("ИмяКолонки");

ЭлементОтбора = ЭлементУО.Отбор.Элементы.Добавить(Тип("ЭлементОтбораКомпоновкиДанных"));
ЭлементОтбора.ЛевоеЗначение = Новый ПолеКомпоновкиДанных("Объект.ТЧ.Реквизит");
ЭлементОтбора.ВидСравнения = ВидСравненияКомпоновкиДанных.Равно;
ЭлементОтбора.ПравоеЗначение = Истина;

ЭлементУО.Оформление.УстановитьЗначениеПараметра("ЦветФона", WebЦвета.Красный);
```

## MCP-инструменты для работы с формами

### Изучение формы
- **`inspect_form_layout`** — получить дерево элементов формы (id, dataPath, свойства). **Всегда вызывать перед модификацией.**
- **`bsl_list_methods`** — структура модуля формы (обработчики событий)
- **`bsl_get_method_body`** — чтение конкретного обработчика

### Модификация формы
- **`edt_validate_request`** — получить `validation_token` (обязательно перед любой мутацией)
- **`mutate_form_model`** — добавление/удаление/перемещение элементов формы:
  - `add_group` — группа, страница, панель
  - `add_field` — поле ввода, флажок, надпись
  - `set_item` — изменение свойств элемента
  - `remove_item` — удаление элемента
  - `move_item` — перемещение в другую группу/позицию
  - `set_form_props` — свойства формы (заголовок, размеры)

### Создание формы
- **`create_form`** — создание новой управляемой формы
- **`apply_form_recipe`** — декларативное создание формы с реквизитами и макетом за один вызов
- **`ensure_module_artifact`** — гарантировать наличие файла модуля формы перед записью кода

### Связь с паттерном ЛМ_ПодготовитьФорму

Для **типовых форм** (модифицируемых через расширения или `#Область Ламода`):
- Элементы добавляются **программно** в `ЛМ_ПодготовитьФорму()`, а не через MCP-инструменты
- MCP-инструменты используются для **изучения** структуры типовой формы (`inspect_form_layout`)

Для **собственных форм** (созданных с нуля):
- Структуру формы создавать через `apply_form_recipe` или `create_form` + `mutate_form_model`
- Модуль формы: `ensure_module_artifact` → `write_file` / `edit_file`
- Проверка: `syntaxcheck`

### Важные правила
- **НЕ редактировать `.mdo` файлы** напрямую через `edit_file`/`write_file` — только MCP-инструменты метаданных
- **Всегда `inspect_form_layout`** перед модификацией формы
- **Всегда `edt_validate_request`** перед мутацией (получить `validation_token`)
